# Release Notes - OpenL MCP Server v1.1.0

**Planned release**: July 2026  
**Works with**: OpenL Studio 6.0.0 or later  
**AI clients**: Claude Code, Claude Desktop, Cursor, VS Code (GitHub Copilot)

OpenL MCP connects your AI assistant to OpenL Studio so you can work with business rules in everyday language — create and change tables, run tests, investigate results, and deploy — without learning the technical details of the platform.

Version 1.1.0 is a major step forward for day-to-day rule work: **more precise table editing**, **smoother test and trace investigation**, **easier project setup**, and **clearer feedback** when something needs your attention.

---

## At a Glance

| What changed | What it means for you |
|--------------|----------------------|
| **Surgical table edits** | Add, remove, or change individual rows, columns, or cells without rebuilding an entire table |
| **Smarter trace investigation** | Ask for a full trace result in one request — the assistant waits for the trace to finish |
| **Clearer error messages** | When an edit fails or a table moves, you get a plain explanation instead of a cryptic error |
| **Project copy and search** | Clone a project in one step; search project files by name or content |
| **Simpler setup for IT** | No custom Docker image to maintain — standard tools your team likely already uses |

**Tool count**: 52 capabilities (12 new since v1.0.0), including 11 fine-grained table-edit actions and the ability to delete a whole table.

> **Note for IT and developers**: This release includes configuration changes (authentication, HTTP connection settings, Docker setup). See [Technical details and upgrade guide](#technical-details-and-upgrade-guide) below.

## What's New for You

### Business Analysts and Rule Authors

**Edit tables with precision** — You can now ask for small, targeted changes instead of replacing whole tables:

- *"Add a column called 'Fleet Discount' after 'Premium' in the Auto Rating table"*
- *"Delete the third row from the Underwriting Rules table"*
- *"Change the value in cell B5 to 0.15"*
- *"Merge the header cells across columns C through E"*

**Work with spreadsheets more naturally** — Multi-column spreadsheet tables can be extended row by row, the way you would in Excel.

**Copy projects faster** — Clone an existing project (rules, tests, settings) under a new name in a single step. The copy shows up in project lists right away.

**Get clearer guidance when something shifts** — If a table moves after an edit (an OpenL Studio behavior), the assistant explains that your change was saved and how to continue — instead of stopping with an unhelpful "not found" message.

**Catch mistakes before they are saved** — Invalid edits (for example, a row with the wrong number of columns) are rejected with a specific explanation, not silently saved with blank cells.

### QA Teams and Testers

**Investigate rule behavior in one conversation** — Traces no longer require you to wait and ask again:

- *"Start a trace for the Premium Calculation table and show me the full decision tree"*
- *"Export the trace so I can share it with the team"*

The assistant waits for the trace to complete and reports progress along the way.

**Find tests and related files quickly**:

- *"Find all test files that mention California"*
- *"Show me everything in the tests folder for auto rating"*

**New investigation guide** — A bundled *openl-trace-investigation* skill helps diagnose unexpected results (wrong premium, null values, rejected claims), including cases where the trace itself looks fine but upstream data is wrong.

**Validate after changes** — Check whether a project still compiles cleanly after edits, with an option to wait until compilation finishes.

### Operations and Release Teams

**Simpler deployment** — Your IT team no longer needs to pull and maintain a custom OpenL MCP Docker image. The server runs via standard `npx` or a generic Node.js container.

**One-command demo stack** — `compose.yaml` can start OpenL Studio and the MCP server together for evaluation or shared environments.

**Deploy from scripts** — The same tool can be run from the command line for automation (listing projects, checking status, deploying) without an AI client — useful for CI/CD pipelines your team may already use.

### Product Owners and Reviewers

**Review project guidance in context** — Project-specific AGENTS.md instructions (how this project expects the AI to work) can be loaded and applied automatically.

**Search across project content** — Find rules, tests, and documentation by keyword without opening files one by one.

**Better feedback loops** — When the assistant makes a mistake, it receives detailed reasons and can correct itself in the same conversation.

## What IT Needs to Do (Summary)

If your organization already runs OpenL MCP v1.0.0, your technical team should plan a short upgrade. **Business users do not need to change how they talk to the assistant** — only the connection setup may change.

| Area | What changed |
|------|--------------|
| **Sign-in** | Username/password login is removed. Use a Personal Access Token from OpenL Studio, or no token for single-user/demo setups. |
| **Remote HTTP connections** | HTTP clients (Cursor, VS Code) connect to `/mcp` with the Streamable HTTP transport — not the older `/mcp/sse` endpoint. |
| **Docker** | Replace the custom `openl-mcp` image with `node:lts-alpine` running `npx -y openl-mcp`. |
| **Claude Desktop** | The `mcp-remote` proxy is no longer used; the server starts directly via `npx`. |

Full step-by-step instructions: [Technical details and upgrade guide](#technical-details-and-upgrade-guide).

Setup help: [Quick Start](../guides/quick-start.md) · [Advanced Guide](../guides/advanced.md) · [Troubleshooting](../guides/troubleshooting.md)

## Known Limitations

- **Some projects must be opened first** — For Git-based repositories, certain operations require the project to be open in OpenL Studio before editing.
- **Large answers are shortened** — Very long responses are trimmed with guidance to request more detail in smaller pieces.
- **Branch-specific project copies** — Cloning onto a named Git branch may not appear in project lists until OpenL Studio re-indexes the repository.
- **Blank new projects** — Always created on the repository's default branch.

## Technical Details and Upgrade Guide

*This section is for developers, administrators, and anyone upgrading from v1.0.0.*

### New and Enhanced Tools

**12 new capabilities** (52 total):

| Category | What's new |
|----------|------------|
| **Table edits (11 tools)** | `openl_append_table_row`/`_column`, `openl_insert_table_row`/`_column`, `openl_delete_table_row`/`_column`, `openl_update_table_row`/`_column`/`_cell`, `openl_merge_table_cells`, `openl_unmerge_table_cells` |
| **Table management** | `openl_delete_table` — remove an entire table |
| **Project files** | `openl_search_project_files` — search by path pattern, extension, type, or content |

Each raw table-edit tool works on any table type, uses 0-based positions, returns the current table ID (and `previousTableId` when the table relocates), and triggers recompilation.

**Also added or improved since v1.0.0** (see [CHANGELOG](../../CHANGELOG.md) for the full list): CLI mode, `openl_create_project`, `openl_project_status`, project file tools, trace tools with server-side waiting, `openl_get_project_agent_context`, and the `openl-trace-investigation` skill.

### CLI Mode

```bash
# Start MCP server (stdio)
npx -y openl-mcp http://localhost:8080

# Run a tool directly (no openl_ prefix)
npx -y openl-mcp http://localhost:8080 list_projects '{"repository":"design"}' --token <PAT>

# Discovery
npx -y openl-mcp --help
npx -y openl-mcp --list-tools
```

See [CLI Guide](../../README.cli.md) and [Advanced Guide — CLI mode](../guides/advanced.md#cli-mode).

### Deployment

```bash
# With Node.js
npx -y openl-mcp http://localhost:8080

# Without Node.js
docker run --rm -i node:lts-alpine npx -y openl-mcp http://host.docker.internal:8080
```

**Docker Compose** (`compose.yaml`) starts OpenL Studio and the MCP server together. The MCP service uses `node:lts-alpine` with `npx` and the `--http` flag; see the file in the repository for the current command (including the nightly `x` prerelease option).

### Breaking Changes

| Change | Action required |
|--------|-----------------|
| **Streamable HTTP** replaces HTTP+SSE | Point HTTP clients at `http://<mcp-host>:3000/mcp` with `"transport": "streamablehttp"`. Remove `/mcp/sse` and `/mcp/messages` configs. |
| **PAT-only authentication** | Replace username/password with `OPENL_PERSONAL_ACCESS_TOKEN` or `--token`. Query-string tokens are no longer accepted. |
| **No custom Docker image** | Use `node:lts-alpine` + `npx` instead of `ghcr.io/openl-tablets/openl-mcp`. |
| **CLI tool names** | Drop the `openl_` prefix in shell scripts (`list_projects`, not `openl_list_projects`). MCP clients still use `openl_` names. |
| **MCP resources removed** | Use equivalent tools (`openl_list_projects`, `openl_project_status`, etc.). |
| **`mcp-remote` removed** | Claude Desktop launches the server directly via `npx`. |
| **Removed env vars / flags** | `OPENL_USERNAME`, `OPENL_PASSWORD`, `OPENL_CLIENT_DOCUMENT_ID`, `--user`, `--password`, `--anonymous`, `--client-document-id` |

### Migration from v1.0.0

**1. HTTP client configuration** (Cursor, VS Code — URL is the MCP server, not OpenL Studio):

```json
{
  "mcpServers": {
    "openl": {
      "url": "http://localhost:3000/mcp",
      "transport": "streamablehttp"
    }
  }
}
```

**2. Personal Access Token**

```bash
export OPENL_PERSONAL_ACCESS_TOKEN=openl_pat_<publicId>.<secret>
```

Generate in OpenL Studio under Admin → Personal Access Tokens.

**3. Resource URI replacements**

| Removed resource | Use instead |
|------------------|-------------|
| `openl://repositories` | `openl_list_repositories` |
| `openl://projects` | `openl_list_projects` |
| `openl://projects/{id}` | `openl_get_project` |
| `openl://projects/{id}/status` | `openl_project_status` |
| `openl://projects/{id}/agents-md` | `openl_get_project_agent_context` |

**4. Table and trace behavior** — No migration needed. Table IDs are tracked across relocations automatically. Trace tools wait for completion by default (`wait: false` restores immediate 409).

### Fixed Issues (Technical)

- Tool failures return detailed `isError` messages instead of generic protocol errors
- Global install (`npm i -g openl-mcp`) launches correctly
- Server starts without credentials for single-user OpenL Studio
- `openl_create_project` clone is atomic and indexed on the default branch
- Pre-write validation on table payloads; RawSource row-width validation
- HTTP mode honors positional URL (`openl-mcp <url> --http`)
